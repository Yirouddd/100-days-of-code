## [665] Non-decreasing Array

**Difficulty**: ⭐️⭐️ Medium  
**Date**: 2026-02-28  
**Tags**: `Greedy Algorithm`  

### Problem Description
Given an array nums with n integers, your task is to check if it could become non-decreasing by modifying at most one element.

### 解题思路

#### 1. 问题分析
非递减数组要求：`nums[i] <= nums[i+1]` 对于所有 `i` 成立。  
当发现一个下降点 `nums[i] > nums[i+1]` 时，我们需要决定如何修改。

#### 2. 关键观察
当发现逆序对 `(nums[i], nums[i+1])` 时，有两种修改策略：

- **策略 A：降低 `nums[i]`**  
  将 `nums[i]` 改为 `nums[i+1]`  
  条件：修改后要保证 `nums[i-1] <= nums[i+1]`  
  即需要检查 `i-1` 位置的情况

- **策略 B：升高 `nums[i+1]`**  
  将 `nums[i+1]` 改为 `nums[i]`  
  条件：修改后要保证 `nums[i] <= nums[i+2]`  
  即需要检查 `i+2` 位置的情况

#### 3. 算法步骤
1. 遍历数组，找到第一个违反非递减的位置 `i`
2. 如果没找到违反的位置，直接返回 `true`
3. 如果找到多个违反位置，返回 `false`
4. 对于唯一违反的位置 `i`，尝试两种修改策略：
   - 如果能通过降低 `nums[i]` 解决，返回 `true`
   - 如果能通过升高 `nums[i+1]` 解决，返回 `true`
5. 否则返回 `false`

#### 4. 边界情况处理
- 当 `i = 0` 时，只需要考虑升高 `nums[1]`
- 当 `i = n-2` 时，只需要考虑降低 `nums[i]`
- 注意数组长度为 1 或 2 的特殊情况

#### 5. 时间复杂度
- 一次遍历：**O(n)**
- 空间复杂度：**O(1)**
